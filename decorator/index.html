Decorator Pattern

What is it?

A design pattern that allows behavior to be added to an object.

Advantages

  • Altnerative to inheritance.
  • Infinite permuations with ease.
  • Other objects of the same class are not affected.

Disadvantages

  • Implementation can be cumbersome in some languages.
  • May be an unfamiliar pattern to peers.
